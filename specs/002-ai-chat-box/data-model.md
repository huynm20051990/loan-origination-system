# Data Model: AI Assistant Chat Box (002-ai-chat-box)

---

## Domain Entities

### ChatSession (Aggregate Root)

Represents a single chat session scoped to a user's page visit.

| Field | Type | Constraints | Notes |
|---|---|---|---|
| `id` | UUID | PK, auto-generated | Surrogate key |
| `userId` | String | NOT NULL, indexed | JWT subject claim from Bearer token |
| `createdAt` | Instant | NOT NULL, auto-set | UTC timestamp |

**Business rules**:
- A session belongs to exactly one user (`userId`). All queries MUST filter by `userId`.
- A session may contain zero or more `ChatMessage` entities.
- Deleting a session cascades to all its messages.

**State transitions**: Created ‚Üí Active (messages added) ‚Üí Deleted (explicit DELETE or cascade)

---

### ChatMessage (Entity within Session)

Represents a single message turn ‚Äî either a user query or an assistant response.

| Field | Type | Constraints | Notes |
|---|---|---|---|
| `id` | UUID | PK, auto-generated | Surrogate key |
| `sessionId` | UUID | FK ‚Üí ChatSession.id, NOT NULL | ON DELETE CASCADE |
| `role` | `MessageRole` (Enum) | NOT NULL | `USER` or `ASSISTANT` |
| `content` | TEXT | NOT NULL | Full message text |
| `timestamp` | Instant | NOT NULL, auto-set | UTC timestamp |

**Business rules**:
- Messages are ordered by `timestamp` ascending within a session.
- `role = USER` messages are created immediately on submission.
- `role = ASSISTANT` messages are persisted only after the SSE stream completes (`done: true`).

---

### MessageRole (Value Object / Enum)

```
USER      ‚Äî message sent by the authenticated user
ASSISTANT ‚Äî message generated by the AI
```

## PostgreSQL Schema (`chat-pg`, port 5436)

Database: `chat-db`  
Init script: `database/init-chat/01.schema.sql`

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE IF NOT EXISTS chat_sessions (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id     VARCHAR(255) NOT NULL,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_chat_sessions_user_id ON chat_sessions(user_id);

CREATE TABLE IF NOT EXISTS chat_messages (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id  UUID NOT NULL REFERENCES chat_sessions(id) ON DELETE CASCADE,
    role        VARCHAR(20) NOT NULL CHECK (role IN ('USER', 'ASSISTANT')),
    content     TEXT NOT NULL,
    timestamp   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_chat_messages_session_id ON chat_messages(session_id);

-- RAG: home listing embeddings (synced from home-service every 15 minutes)
CREATE TABLE IF NOT EXISTS chat_home_embeddings (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    content     TEXT,
    metadata    JSONB,
    embedding   vector(768)
);
CREATE INDEX chat_home_embeddings_embedding_idx
    ON chat_home_embeddings USING hnsw (embedding vector_cosine_ops);
```

## Cassandra Schema (Spring AI Chat Memory)

Keyspace: `chat_memory`  
Cluster: shared `cassandra` container (port 9042, datacenter1)  
Init script: `database/init-chat-cassandra/init.cql`

```cql
CREATE KEYSPACE IF NOT EXISTS chat_memory
  WITH replication = {'class': 'SimpleStrategy', 'replication_factor': 1};

CREATE TABLE IF NOT EXISTS chat_memory.messages (
    session_id  UUID,
    message_id  UUID,
    role        TEXT,
    content     TEXT,
    timestamp   TIMESTAMP,
    PRIMARY KEY (session_id, message_id)
) WITH CLUSTERING ORDER BY (message_id ASC);
```

**Usage**: Spring AI `CassandraChatMemory` reads/writes this table scoped by `sessionId`. Maximum 20 message turns retained in context (Spring AI window size configuration).

## Validation Rules

| Field | Rule | Enforced At |
|---|---|---|
| `ChatMessageRequestDTO.content` | `@NotBlank` ‚Äî must not be empty or whitespace | DTO boundary (Spring Validation) |
| `sessionId` path param | Must be a valid UUID | Controller input validation |
| Session ownership | `session.userId` must equal JWT subject | `ChatApplicationService` before any operation |
| `role` | Must be `USER` or `ASSISTANT` | DB CHECK constraint + Enum deserialization |
| `embedding` dimension | Must be 768 | pgvector column definition |
